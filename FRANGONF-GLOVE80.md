# frangonf Glove80 — Custom Layout

Custom ZMK firmware for Glove80 with **64-layer support** and dual macOS/Linux architecture.

---

## Quick Build Reference

```bash
# Build combined firmware (both halves)
nix-build -A glove80_combined
# Output: ./result/glove80.uf2

# Flash: Copy result/glove80.uf2 to GLV80LHBOOT and GLV80RHBOOT
```

> **First time?** Install Nix: `curl -L https://nixos.org/nix/install | sh`

## What We Changed

### Core Modifications

| Change               | Files                                          | Purpose                                                            |
| :------------------- | :--------------------------------------------- | :----------------------------------------------------------------- |
| **64-layer support** | `app/include/zmk/keymap.h`, `app/src/keymap.c` | `uint32_t` → `uint64_t` layer state to support massive feature set |
| **32-bit fixes**     | `app/src/combo.c`, `app/src/input_listener.c`  | Fix overflow in high layers                                        |
| **RGB underglow**    | Merged from darknao fork                       | Per-layer RGB indicators                                           |

### Massive Behavior Additions

We have significantly expanded the capabilities of ZMK beyond standard keymaps, adding ~7,000 lines of behavior definitions:

- **World Characters** (`behaviors/world-characters.dtsi`): Comprehensive support for international symbols, currencies, and accented characters using Unicode (macOS) and Compose sequences (Linux).
- **Emoji Support** (`behaviors/emoji.dtsi`): Extensive library of emoji accessible directly from the keyboard.
- **Bilateral Enforcement**: Custom Home Row Mod implementation in `behaviors/home-row-mods.dtsi` to prevent accidental activations.

### Layout Architecture

```
Layers 0-31:   macOS (Cmd-based shortcuts, hex Unicode input)
Layers 32-63:  Linux (Ctrl-based shortcuts, Compose sequence input)
```

Each OS has a **complete mirror** of all layer types:

- Base layouts (QWERTY, Enthium, Engrammer, etc.)
- Finger layers (home row mod activation)
- Function layers (cursor, numpad, symbols)
- Mouse layers
- Special layers (Gaming, Magic)

        └── ...                      # All layer types × 2 OS

## How to Modify the Layout

### Change a key binding

Edit the layer file in `includes/layers/`.

```c
// In base-layers.dtsi, find layer_QWERTY:
layer_QWERTY {
    bindings = <
        &kp ESC  &kp N1  ...  // Edit here
    >;
};
```

### Add a new behavior

1. Create in `includes/behaviors/` or edit existing
2. Include it in `includes/behaviors.dtsi`

### Change home row mod timing

Edit `includes/behaviors/definitions.dtsi`:

```c
#define HRM_PRIOR_IDLE 150  // Adjust timing here
#define HRM_TAPPING_TERM 280
```

### Toggle bilateral enforcement

```c
#define ENFORCE_BILATERAL        // macOS (comment to disable)
#define ENFORCE_BILATERAL_LINUX  // Linux
```

### Switch base layout

The default is QWERTY (layer 0). Other layouts available:

- Enthium (1), Engrammer (2), Engram (3), Dvorak (4), Colemak (5), ColemakDH (6)

## Key Design Decisions

| Feature              | Approach              | Rationale                                                                                              |
| :------------------- | :-------------------- | :----------------------------------------------------------------------------------------------------- |
| **Base keymap**      | **Sunaku v42-rc6**    | Chosen for its modularity, well-tested home row mods, and community support.                           |
| **64 Layers**        | `uint64_t` adaptation | Needed to support the dual-OS architecture where every layer is duplicated for macOS and Linux.        |
| **Unicode Strategy** | Hybrid (Hex/Compose)  | macOS supports Hex Input reliably; Linux is best with Compose Key sequences. We support both natively. |
| **Home row mods**    | Bilateral Enforcement | Reduces misfires by requiring opposite-hand modifiers, critical for fast typing.                       |
| **Find/Replace**     | Composable Shortcuts  | Unified shortcuts (Cmd/Ctrl+F) abstract away the OS differences.                                       |
